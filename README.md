# StyleSync-Novel

一个面向小说创作场景的 AI 产品原型，用于验证 `LLM + 风格分析 + RAG + 工作流` 在风格模仿、设定补全、章节规划和正文生成中的落地方式。

它当前的定位不是线上产品，而是一个可本地运行、可演示、可测试、可继续迭代的 MVP，适合：

- AI 产品经理 / AI 应用方向求职作品集
- AIGC 小说创作工具原型展示
- 本地可控的小说生成与创作辅助实验

## 项目定位

这个项目关注的不是“做一个通用聊天壳”，而是把小说创作拆成一条更可控的工作链路：

1. 参考文本分析
2. 风格与设定提取
3. 设定补全
4. 章节大纲生成
5. 正文生成
6. 工作台微调与人工干预
7. 文本校验与后续迭代

其中，中间的对话区域已经被重新定位为“工作台”：

- 支持快捷提示词模板
- 支持从知识库注入内容到输入框
- 支持从正文编辑区注入内容到输入框
- 支持把 `f5b` 的生成提示词注入工作台进行人工改写

这使得产品不再只是“结构化流水线”或“普通聊天”，而是形成了“稳定生成 + 灵活修正”的组合能力。

## 适用用户

- 原创小说作者：希望提升连载效率，同时保留对设定和文风的控制
- 同人创作者：希望在尽量保留原著风格和设定的前提下生成衍生内容
- AI 产品 / AIGC 方向面试官与测试者：希望快速理解产品链路、交互设计和原型能力

## 当前 MVP 的核心能力

### 1. 风格分析与知识准备

- 参考文本统计与风格特征提取
- 高频词、关键词、专属词汇抽取
- 世界观设定与角色卡生成
- 项目级记忆库 / RAG 相关能力预留与接入

### 2. `f5a` 章节大纲生成

`f5a` 面向“先规划再写作”的场景，负责把章节目标转成结构化大纲。

当前支持：

- 输入章节概述与章节定位
- 设定视角、节奏、篇幅、结构阶段
- 结合项目上下文生成章节大纲
- 可作为后续 `f5b` 正文生成的上游输入

它更像一个“结构规划节点”，解决的是“这一章要写什么、怎么铺、如何推进”的问题。

### 3. `f5b` 正文生成

`f5b` 面向“从设定与大纲到正文”的场景，负责基于已有项目知识和章节信息生成正文。

当前支持：

- 基于项目设定和章节名发起正文生成
- 正文生成完成后同步到内容编辑区
- 可选在执行时把 `f5b` 提示词注入工作台输入框
- 可选“只注入提示词，不生成小说”

这让 `f5b` 既能作为自动生成节点，也能作为 prompt 导出与人工二次改写的入口。

### 4. 工作台

工作台是本轮原型迭代后最重要的交互层，用于承接人工微调。

当前支持的快捷动作包括：

- 修改角色卡
- 修改世界观
- 润色章节大纲
- 生成正文
- 修改小说正文

当前支持的注入来源包括：

- 知识库内容注入工作台
- 小说正文注入工作台
- `f5b` 提示词注入工作台

设计目标是让用户可以在结构化流程之外，对局部内容进行追问、改写、补全和试验，而不需要每次都重跑整条流水线。

## 关键交互设计

### 工作台与知识库的关系

- 知识库负责存储结构化创作资产
- 工作台负责基于这些资产进行微调、追问和生成

典型链路：

1. 在知识库中打开角色卡 / 世界观 / 大纲
2. 点击“注入工作台”
3. 工作台自动写入对应模板与内容
4. 用户继续补充、追问或直接发送

### 工作台与 `f5b` 的关系

`f5b` 不再只是“点一下直接生成”的黑盒节点，还支持：

- 先导出本次生成用的提示词
- 注入到工作台输入框
- 人工观察、修改、追问
- 再决定是否继续生成正文

这对作品展示很重要，因为它能体现“可控生成”而不是单纯 API 调用。

## 任务队列说明

任务队列当前将“停止任务”和“删除记录”分开处理：

- `终止`
  - `终止最近`
  - `终止全部`
  - `终止勾选`
- `清除`
  - `清除最远`
  - `清除全部`
  - `清除勾选`

当前真实语义是：

- `终止`：停止后台任务进程，并把任务状态标记为 `cancelled`
- `清除`：删除任务栏中的任务记录

注意：

- 目前 `终止` 不等于“全局回滚本次产物”
- 也就是说，终止任务后，不保证自动清理所有可能已经写入的中间结果或半成品
- 这部分后续如果继续工程化，可进一步改造成“临时文件 + 成功提交 + 终止清理”的机制

## 技术方案

- 后端：FastAPI
- 前端：原生 HTML + JavaScript + Vue 组合式 API（浏览器端使用）
- 本地文本处理：Jieba、词频统计、文件读写
- 向量检索：FAISS + `BAAI/bge-m3`
- 大模型：DeepSeek 系列模型
- 运行方式：本地工作区 + 云端 LLM API

更完整的设计说明见 [docs/technical-design.md](docs/technical-design.md)。

## 快速开始

### 1. 准备环境

建议使用 Python `3.11` 或 `3.12`。

```powershell
py -3.11 -m venv venv
venv\Scripts\activate
python -m pip install -r requirements.txt
```

### 2. 配置 `.env`

先复制模板：

```powershell
Copy-Item .env.example .env
```

至少需要填写：

```env
DEEPSEEK_API_KEY=your_deepseek_key
SILICONFLOW_API_KEY=your_siliconflow_key
```

可选：

```env
DEFAULT_CHAT_MODEL=deepseek-chat
DEFAULT_EMBEDDING_MODEL=BAAI/bge-m3
```

### 3. 启动服务

```powershell
python style_imitation_code/main.py
```

启动后访问：

- [http://127.0.0.1:8000](http://127.0.0.1:8000)

## 推荐体验路径

如果你是第一次体验这个项目，建议按下面顺序：

1. 选择参考文本，执行风格提取相关脚本
2. 创建项目并查看知识库内容
3. 使用 `f4a` 做设定补全
4. 使用 `f5a` 生成章节大纲
5. 使用 `f5b` 生成正文
6. 把知识库内容、正文或 `f5b` 提示词注入工作台继续微调

- 项目创建与章节读写

当前测试文件包括：

## 当前限制

- 当前是本地原型，不是线上托管服务
- 生成质量依赖参考文本质量、提示词设计和召回效果
- 任务系统当前更偏向 MVP，`终止` 仅停止任务，不保证全量清理已写产物
- 当前测试以接口和最小回归为主，还没有形成完整的生成质量评估体系

## 作为求职作品时最值得展示的点

- 不是单纯聊天壳，而是结构化创作工作流
- `f5a` 和 `f5b` 分工清晰：前者负责规划，后者负责正文生成
- 工作台承担“人工微调层”的角色，能承接知识库、正文和 prompt 注入
- 支持在自动生成之外保留人工控制与调参空间
