StyleSync-Novel 系统说明文档

一、 项目概述

本项目是一个基于大语言模型（Large Language Model）与检索增强生成（RAG）技术的自动化小说续写与风格模仿系统。系统通过本地自然语言处理算法与大模型API的协同工作，旨在解决长文本生成过程中常见的角色设定偏离、上下文记忆遗忘以及行文风格不一致等工程问题。项目主要服务于同人小说衍生创作与原创小说连载辅助生成。

二、 技术架构

本系统采用前后端分离与多进程任务调度的架构设计，核心技术栈涵盖以下维度：

1. 后端路由与并发调度：使用 FastAPI 构建接口服务，采用 asyncio 提供进程级别的并发锁控制（通过 Semaphore 分离计算密集型任务与网络 I/O 密集型任务）。
2. 前端交互与视图渲染：基于 Vue 3 采用响应式数据绑定构建单页面应用（SPA）。通过 Server-Sent Events (SSE) 协议实现大模型流式输出的实时解析与 Markdown 动态渲染，并结合 DOMPurify 防御潜在的 XSS 注入风险。
3. 自然语言处理与向量库：集成 Jieba 进行本地中文分词与词性分析；接入硅基流动（SiliconFlow）API 调用 BAAI/bge-m3 模型进行文本向量化编码；集成 FAISS 构建本地离线向量数据库，实现高维空间相似度检索。
4. 大语言模型接口：接入 DeepSeek V3（标准对话模型）与 DeepSeek R1（深度推理模型），负责核心逻辑推演与文本生成。
5. 安全防护与并发控制：在文件读写层封装了带有指数退避与紧急备份策略的原子写入（Atomic Write）机制，防范 Windows 环境下高频并发导致的文件句柄独占锁死（WinError 32）。生成链路层内置正则表达式敏感词巡检与阈值熔断机制。
6. 版本与存储控制：依赖本地文件系统进行物理目录级的状态管理，所有生成设定均以纯文本或 Markdown 格式落盘，并使用 Git 进行代码版本迭代控制。

三、 核心工作流与已实现功能

系统已完整实现功能一至功能七（除 f6 外）的闭环执行逻辑：

1. 文本特征与物理指标提取 (f0, f1a, f1b)
统计分析：读取全量参考文本，计算段落长度方差、长短句分布、标点符号密度、词性分布以及基础词汇丰富度（对数 Type-Token Ratio）。
深层特征：对原文样本切片并输入大模型，提取叙事节奏、视角倾向、心理描写途径及修辞手法特征，生成系统级文风约束文件。

2. 词库清洗与设定实体提取 (f2a, f2b, f3a, f3b, f3c)
高频词过滤：利用动态算法限制高频词提取上限，避免长文本导致的计算资源溢出。
设定信息抽离：针对提取的核心实体，在全量文本中执行高相关度检索。生成结构化的世界观设定与独立的角色信息卡。

3. 剧情压缩与双轨 RAG 记忆系统构建 (f4b, f4c)
原著背景 RAG (f4b)：通过正则表达式识别章节节点，将原著切分为物理文本块并提取核心实体，生成局部伪摘要。将其向量化存入 FAISS 索引，建立摘要至原始长文本块的 JSON 映射关系。
工程动态上下文 RAG (f4c)：系统支持读取当前创作项目下已生成的历史章节正文，动态构建前文记忆 RAG 库。在生成新章节时，双轨并行检索原著设定与前文剧情，确保上下文无缝衔接。

4. 章节大纲与流式正文生成 (f5a, f5b)
动态约束注入：在生成新章节大纲前，系统读取用户输入的剧情简述，通过字符串匹配算法动态筛选并仅加载相关的角色信息卡，降低大模型上下文窗口的参数冗余度。
流式异步生成：依据前置生成的世界观设定、章节大纲及前文检索上下文，组装最终的系统级与用户级提示词。调用大模型 API 执行正文生成，前端接收 SSE 数据流并进行实时页面交互渲染与本地文件追加写入。

5. 文本多维度交叉校验 (f7)
闭环验证逻辑：针对生成的正文与大纲，通过物理指标（如字数下限）、结构指标（Markdown 节点完整性）与合规指标（本地负面词库碰撞）进行交叉校验，输出综合评估分数与反馈指导。

四、 用户界面与操作指南

系统启动后，通过浏览器访问本地 8000 端口进入 Web 工作台。工作台采用双栏布局，左侧为全局控制台，右侧为核心业务区。

1. 全局控制台
任务队列监控：实时展示异步任务的执行状态、耗时与 Token 消耗量。支持底层标准输出流式日志的截断回传。
安全与参数配置：支持动态配置 API 密钥、全局禁用词列表、重试上限与敏感词容忍度。
多线程会话管理：支持创建多个独立的对话进程，调整 Temperature 与 Top P 参数，管理独立上下文。

2. 内容编辑器
提供工程级别的目录树管理。支持文本的实时自动保存与防抖机制，提供 Markdown 渲染视图，并支持将生成结果导入物理文件。

3. 自动化工作流
文风提取流水线：支持上传参考原著，提供全自动提取与单步调试执行两种模式。
小说创作工程：支持基于参考文风建立原创或同人项目。输入章节名与剧情简述后，系统自动执行前文记忆入库、大纲推演、文本校验、正文流式生成的自动化闭环。

4. 知识库管理
提供基于 Web 的可视化 Markdown 编辑器，允许用户在不直接操作宿主机文件系统的情况下，查阅并覆写底层世界观设定、专属词库、大纲结构与角色面板数据。

六、 部署与运行环境配置

本系统的环境搭建需满足 Python 3.9 及以上语言版本要求。

1. 配置环境变量
在项目根目录（D:\StyleSync-Novel\）创建 .env 配置文件，声明所需的模型调用凭证：
DEEPSEEK_API_KEY="在此处填入DeepSeek的API密钥"
SILICONFLOW_API_KEY="在此处填入硅基流动的API密钥"

2. 构建虚拟环境与安装依赖
在系统终端中执行以下指令，隔离项目运行环境并配置依赖：
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt

3. 启动后端服务
在已激活虚拟环境的终端中运行主入口文件：
python style_imitation_code\main.py
服务启动后，本地进程默认监听 0.0.0.0:8000 端口。

七、 核心操作流程示例

1. 文本预处理：在“自动化工作流”面板上传目标参考文本（如 AI 生成的无版权基准文本）。
2. 特征提取：选择“一键全自动提取”模式，系统依次执行 f0 至 f4b 脚本，完成向量库初始化与设定文件的生成。
3. 工程初始化：在小说创作工程模块，输入新项目名称并选择对应的参考文风库，执行“创建新工程”。
4. 闭环生成：在目标项目下，输入目标章节名与核心剧情简述，勾选“开启自动记忆构建”与“开启全局自动校验”，点击“执行创作脚本”。系统将自动完成前文检索、大纲构建、正文生成与质量校验。

八、 待实现功能与系统优化规划

1. 剧情方向自动化推演 (f6)：计划开发遍历历史前文摘要数据集的查询算法，定向提取处于休眠状态的线索实体，结合大语言模型推演多分支可能的剧情走向。
2. 算法权重与版本控制优化：计划在词汇提取阶段引入 TF-IDF 与 TextRank 算法。通过与开源双语语料库对比筛除低价值数据，并为高价值词汇分配浮点数学权重。计划对角色信息卡建立带章节锚点的时间线版本控制机制。
