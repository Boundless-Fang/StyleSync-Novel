# StyleSync-Novel

一个面向小说创作场景的 AI 产品原型。  
项目把创作流程拆成 `风格分析 -> 设定补全 -> 章节大纲 -> 正文生成 -> 工作台微调` 的可控链路，用来验证 `LLM + 风格分析 + RAG + 工作流` 在长文本创作中的产品化落地方式。

这个项目当前只是一个放在 GitHub 上展示、获取测试反馈、继续迭代的 MVP，现阶段目标不是部署上线。

## 项目亮点

- 采用结构化创作链路，强调“先规划，再生成，再人工干预”
- 有真实可运行的前后端原型，不只是静态页面或概念文档
- 把中间对话区重新定义为“工作台”，承接 prompt 注入、知识库注入和正文微调
- 同时提供接口测试、技术文档、评估文档和演示案例，适合作为 AI 产品经理 / AI 应用方向求职作品

## 项目背景与应用场景

在小说创作与风格模仿场景中，创作者通常会遇到以下问题：

- 角色设定容易漂移
- 长文本创作中上下文容易遗忘
- 多轮生成后文风不稳定
- 想改局部内容时，不希望每次都重跑整条链路

StyleSync-Novel 的思路是把创作拆成几个更可控的阶段，并在自动生成与人工修改之间加入一个可操作的工作台，而不是把结果完全交给一次性生成。

## 核心链路

项目当前的主展示链路如下：

1. 导入参考文本，做风格分析与词汇/设定提取
2. 使用 `f4a` 做世界观或角色设定补全
3. 使用 `f5a` 生成章节大纲
4. 使用 `f5b` 基于设定与大纲生成正文
5. 把知识库内容、正文或 `f5b` prompt 注入工作台继续微调

其中：

- `f0-f5b` 是当前 MVP 的稳定主链路
- `f6/f7` 是预留中的 preview 能力，目前仍属实验功能且尚未完成，不属于当前主展示重点

## 适用对象

- AI 产品经理 / AI 应用方向招聘方：适合快速了解项目的产品拆解方式、原型完成度和迭代思路
- AIGC 创作工具测试者：可以快速体验“风格分析 + 结构化生成 + 人工微调”的链路
- 对小说生成、风格模仿、RAG 创作辅助感兴趣的开发者：可以直接查看实现方式和 API 设计

## 当前 MVP 的核心能力

### 1. 风格分析与知识准备

- 参考文本统计与风格特征提取
- 高频词、关键词、专属词汇抽取
- 世界观设定与角色卡生成
- 项目级记忆库 / RAG 能力接入

### 2. `f5a` 章节大纲生成

`f5a` 面向“先规划再写作”的场景，负责把章节目标转成结构化大纲。

当前能力包括：

- 输入章节概述与章节定位
- 设置视角、节奏、篇幅、结构阶段
- 结合项目上下文生成章节大纲
- 作为后续 `f5b` 正文生成的上游输入

### 3. `f5b` 正文生成

`f5b` 面向“从设定与大纲到正文”的场景，负责基于已有项目知识和章节信息生成正文。

当前能力包括：

- 基于项目设定和章节名发起正文生成
- 正文生成完成后同步到内容编辑区
- 可选把 `f5b` 提示词注入工作台输入框
- 可选“只注入提示词，不生成正文”

### 4. 工作台

工作台是这版原型中最重要的交互层，用于承接人工干预。

当前能力包括：

- 角色卡修改
- 世界观修改
- 章节大纲润色
- 正文生成与正文改写
- 知识库内容 / 正文 / `f5b` prompt 注入

它的价值主要体现在给结构化工作流提供一个人工可控的中间层，而不只是承担聊天入口。

## 项目状态

项目目前处于以下状态：

- `MVP`
- 本地运行，不是线上托管服务
- 已有前端原型、FastAPI 后端、工作流脚本、接口测试和技术文档
- 主展示重点是 `f0-f5b + 工作台`
- 生成质量仍然依赖参考文本质量、prompt 设计和召回效果

当前限制：

- 暂未形成完整的自动化生成质量评测体系
- 任务终止不等于全量回滚已写入产物
- 外部测试者仍需本地配置环境

## 项目结构

```text
StyleSync-Novel/
├─ style_imitation_code/
│  ├─ api/                  # FastAPI 路由、任务调度、项目接口
│  ├─ core/                 # LLM、RAG、配置与通用工具
│  ├─ frontend/             # 前端页面与交互逻辑
│  ├─ scripts/              # f0-f7 工作流脚本（其中 f6/f7 为实验中的预留节点）
│  └─ main.py               # 服务启动入口
├─ docs/                    # 技术设计、评估与演示文档
├─ tests/                   # 接口测试与本地测试工具
├─ dictionaries/            # 词库与可扩展配置
├─ reference_novels/        # 运行后使用：参考文本输入目录
├─ text_style_imitation/    # 运行后生成：风格分析与索引产物
├─ novel_projects/          # 运行后生成：项目正文、知识库与中间结果
├─ text_testing_code/       # 本地测试代码目录
├─ requirements.txt
└─ README.md
```

其中 `reference_novels/`、`text_style_imitation/`、`novel_projects/` 等目录主要承载运行期数据，公开仓库中通常不提交其具体内容。

## 快速开始

### 1. 准备环境

建议使用 Python `3.11` 或 `3.12`。

```powershell
py -3.11 -m venv venv
venv\Scripts\activate
python -m pip install -r requirements.txt
```

### 2. 配置 `.env`

先复制模板：

```powershell
Copy-Item .env.example .env
```

至少需要填写：

```env
DEEPSEEK_API_KEY=your_deepseek_key
SILICONFLOW_API_KEY=your_siliconflow_key
```

可选：

```env
DEFAULT_CHAT_MODEL=deepseek-v4-flash
DEFAULT_EMBEDDING_MODEL=BAAI/bge-m3
```

### 3. 启动服务

```powershell
python style_imitation_code/main.py
```

启动后访问：

- <http://127.0.0.1:8000>

## 生成效果评估

这个项目目前不仅关注链路能否跑通，也希望逐步形成一个可复用的 MVP 评估闭环，用于记录结果质量和后续迭代方向。

当前建议重点评估：

- `f5a` 章节大纲的结构完整性、设定一致性、节奏可控性
- `f5b` 正文生成的风格贴合度、章节可读性、人工修改成本
- 工作台介入后，是否能有效降低返工成本

详细评估维度、评分表和记录模板见 [docs/evaluation.md](docs/evaluation.md)。
