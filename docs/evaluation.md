# StyleSync-Novel 生成效果评估说明

## 1. 文档目标

这份文档用于回答两个问题：

1. 这个原型生成出来的结果到底“好不好”？
2. 作为 AI 产品原型，它是如何被测试、比较和迭代的？

项目当前不是线上产品，因此评估重点不是大规模自动化 benchmark，而是面向 MVP 的“小样本、可复现、可解释”评估。

## 2. 评估对象

当前优先评估两类核心产物：

- `f5a`：章节大纲生成结果
- `f5b`：章节正文生成结果

说明：`f6/f7` 目前仍处于实验与预留阶段，暂不纳入当前 MVP 的正式评估主范围。

辅助观察对象包括：

- 风格分析与设定抽取结果是否可复用
- 工作台人工干预后，是否能降低二次修改成本

## 3. 评估原则

- 优先评估“是否可用”，再评估“是否惊艳”
- 优先评估“产品链路是否成立”，再评估“单次文案质量”
- 不只看最终文本，也看中间产物是否有助于后续步骤
- 记录人工修改成本，而不是假设模型一次生成就能完美完成

## 4. `f5a` 评估维度

### 4.1 结构完整性

看生成结果是否回答了“这一章写什么、怎么推进、如何收束”。

建议评分：

- `1` 分：结构混乱，无法直接使用
- `3` 分：结构基本完整，但需要人工补关键环节
- `5` 分：结构清晰，可直接作为正文生成输入

### 4.2 设定一致性

看大纲是否遵守已有角色、世界观、剧情前置条件。

重点观察：

- 角色关系是否跑偏
- 世界观设定是否冲突
- 前文事实是否被遗忘

### 4.3 节奏可控性

看大纲是否体现节奏设计，而不是简单堆情节点。

重点观察：

- 开头是否能快速进入章节目标
- 中段是否有推进而不是重复
- 结尾是否形成钩子、转折或收束

### 4.4 可执行性

看 `f5a` 的结果能否顺利交给 `f5b`，以及是否方便人工继续微调。

重点观察：

- 是否能直接进入正文生成
- 是否适合注入工作台改写
- 是否便于存入章节结构资产

## 5. `f5b` 评估维度

### 5.1 风格贴合度

看正文是否接近参考文本的叙事风格、语言质感和表达习惯。

重点观察：

- 句式和节奏是否接近目标风格
- 词汇选择是否有参考文本特征
- 是否只是“表面像小说”，而不是真正贴近目标风格

### 5.2 设定一致性

看正文是否遵守角色、世界观、章节目标和大纲约束。

重点观察：

- 人设是否稳定
- 世界规则是否自洽
- 是否出现明显事实冲突

### 5.3 章节可读性

看正文是否像一个可以继续编辑的小说章节，而不是松散片段。

重点观察：

- 段落组织是否自然
- 场景衔接是否顺畅
- 对话、动作、描写是否平衡

### 5.4 人工修改成本

看生成结果离“可接受版本”还差多少人工工作。

建议使用三档记录：

- `低`：只需局部润色
- `中`：需要重写部分段落或补逻辑
- `高`：只能保留少量素材，需大幅返工

## 6. 推荐测试方法

当前建议使用 3 组小样本案例：

1. 同人风格模仿案例
2. 原创设定补全案例
3. 从 `f5a` 到 `f5b` 的完整章节案例

每组案例至少保留以下记录：

- 输入材料
- 使用了哪些工作流节点
- 中间产物截图或摘要
- 最终输出节选
- 评估打分
- 主要问题
- 如何通过工作台或 prompt 调整

## 7. 建议评分表

| 维度 | 说明 | 分值 |
| --- | --- | --- |
| 结构完整性 | 章节结构是否清晰可执行 | 1-5 |
| 设定一致性 | 是否遵守人设、世界观、前文事实 | 1-5 |
| 风格贴合度 | 是否接近目标风格 | 1-5 |
| 章节可读性 | 是否像可继续编辑的小说正文 | 1-5 |
| 人工修改成本 | 修改工作量高低 | 低 / 中 / 高 |

可以把总分作为观察指标，但更重要的是记录问题类型和修正方式。

## 9. 对求职展示的意义

这份评估文档的价值不只是“测试说明”，更重要的是体现出：

- 你不是只会调用模型，而是在设计评估闭环
- 你知道 AI 产品不能只看 demo，要看稳定性与可用性
- 你能把生成质量问题拆成可观察、可讨论、可迭代的指标
