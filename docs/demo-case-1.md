# Demo Case 1：从参考风格到章节正文

## 1. 案例目标

这个案例用于演示 StyleSync-Novel 的一条典型主链路：

`参考文本分析 -> 设定补全 -> 章节大纲 -> 正文生成 -> 工作台微调`

说明：本案例只覆盖当前 MVP 的稳定主链路，不包含 `f6/f7` 两个仍在实验中的预留节点。

它适合：

- GitHub 仓库展示
- 面试时口头讲解
- 录制 2-3 分钟产品演示视频

## 2. 演示问题

如果用户希望在保留目标文风和设定约束的前提下，快速产出一个“可继续编辑”的小说章节，这条链路是否成立？

## 3. 输入准备

### 3.1 参考材料

- 选择一篇或一组参考文本
- 要求参考文本风格相对稳定，避免来源过杂

### 3.2 项目类型

- 同人模式：强调风格还原和角色一致
- 原创模式：强调世界观搭建和持续续写

### 3.3 章节目标

建议提前准备一段清晰的章节 brief，例如：

> 主角第一次进入陌生空间，先产生误判，再通过一次冲突确认环境规则，最后留下一个新的悬念。

## 4. 推荐演示流程

### Step 1：展示参考文本分析结果

重点展示：

- 风格特征
- 高频词与专属词汇
- 世界观 / 角色设定资产

讲解重点：

- 这一步不是直接生成正文，而是在准备后续可复用的创作资产

### Step 2：展示 `f4a` 设定补全

重点展示：

- 世界观补全
- 角色卡补全

讲解重点：

- 先补设定，再写章节，比直接从空白 prompt 起步更可控

### Step 3：展示 `f5a` 章节大纲

重点展示：

- 章节目标
- 情节推进
- 节奏安排

讲解重点：

- `f5a` 的价值是把“先规划再写作”显式产品化

### Step 4：展示 `f5b` 正文生成

重点展示：

- 大纲如何进入正文生成
- 结果如何同步到正文编辑区

讲解重点：

- `f5b` 不只是生成器，也是上游约束被执行的结果窗口

### Step 5：展示工作台人工干预

建议演示一种简单但有辨识度的操作：

- 把 `f5b` 提示词注入工作台
- 或把正文片段注入工作台进行改写

讲解重点：

- 这个产品不是“要么全自动，要么手工重来”
- 它提供的是结构化生成与人工微调之间的过渡层

## 5. 建议展示素材

为了让演示更有说服力，建议至少准备以下内容：

- 输入 brief 截图
- `f5a` 大纲截图
- `f5b` 正文截图
- 工作台微调截图
- 最终结果节选

## 6. 推荐讲解话术

### 产品视角

这个原型不是单纯的聊天界面，而是把小说创作拆成几个更可控的步骤，每一步都尽量输出可复用资产，而不是只追求“一次生成到位”。

### AI 能力视角

模型不是孤立工作的，前面有风格分析和设定准备，后面有工作台作为人工干预层，所以整条链路比单轮 prompt 更稳定。

### 求职展示视角

这个案例最能体现的，不是“我会写代码调用模型”，而是“我能把一个 AI 场景拆成可验证、可迭代、可展示的产品流程”。

## 7. 演示后可复盘的问题

演示结束后，可以从下面几个角度继续记录：

- 哪一步最能提升可控性
- 哪一步最容易出现质量波动
- 哪些问题适合继续靠 prompt 修
- 哪些问题需要后续做更明确的产品约束

录完 demo 或完成测试后，建议把本案例的评分记录补到 [evaluation.md](./evaluation.md) 中，形成“演示素材 + 评估记录”的闭环。
